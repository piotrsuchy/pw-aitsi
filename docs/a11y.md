# Accessibility (WCAG) Analysis

The Local Archive is designed to meet WCAG 2.1 AA standards, ensuring that users with varying visual, motor, and cognitive abilities can use the application effectively.

## Verified Techniques & Implementation

### 1. Semantic HTML Structure

The application heavily utilizes semantic HTML5 elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<form>`).

- **Why it matters:** Screen readers use semantic landmarks to map out a page, allowing visually impaired users to jump directly to the main content or navigation.

### 2. Skip Link (`.skip-link`)

A hidden link (`<a href="#main-content" className="skip-link">Skip to content</a>`) is positioned off-screen. It becomes visible only when a keyboard user presses the `Tab` key.

- **Why it matters:** Motor-impaired users who rely on keyboards can bypass repetitive navigation links and jump straight to the photo grid or search form.

### 3. Keyboard Navigation & Focus Rings

All interactive elements (buttons, links, inputs) have clearly defined focus rings (`focus-visible:outline-[var(--primary)]`).

- **Why it matters:** Users who navigate without a mouse must have a clear visual indicator of which element they are currently interacting with.

### 4. Contrast & Themes

The default color palette was mathematically verified to meet WCAG AA minimum contrast ratio thresholds (4.5:1 for regular text, 3:1 for large text).

- **Theme Switcher:** The application supports Light, Dark, and a dedicated **High Contrast** mode designed for low-vision users.
- **Verification:** Tools like `axe-core` were used to ensure colors like `--muted-foreground` pass the 4.5:1 ratio against their backgrounds.

### 5. ARIA Labels and Roles

Wherever visual context cannot be parsed by machines, ARIA properties are used.

- Example: The navigation list uses `role="list"` and `aria-label="Main navigation"`.
- Example: Decorative images (such as photo thumbnails where the title is already written in the DOM underneath) use `alt=""` to prevent screen readers from reading the exact same text twice (fixing `image-redundant-alt` errors).

## Automated Verification Tools

To ensure ongoing compliance, the project integrates `@axe-core/cli` into the testing suite.

**How it is verified:**

- A developer runs the local server (`npm run dev`).
- In a separate terminal, the developer executes `npm run test:a11y`.
- The CLI tool audits the rendered DOM in a headless browser against the `wcag2a` and `wcag2aa` rulesets, instantly highlighting any contrast failures, missing labels, or invalid HTML structures.
